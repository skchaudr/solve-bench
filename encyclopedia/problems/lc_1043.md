# lc_1043: Partition Array for Maximum Sum

**Topic**: [Dynamic Programming](../Dynamic_Programming.md)

**Empirical Time Complexity**: `O(N * K)`

**Empirical Space Complexity**: `O(N)`

## Notes

DP array takes O(N) space. Inner loop runs up to K times, leading to O(N * K) time complexity. With K=50, empirical time scales linearly with N.

## Solution

```python
from typing import List

class Solution:
    def maxSumAfterPartitioning(self, arr: List[int], k: int) -> int:
        n = len(arr)
        dp = [0] * (n + 1)
        for i in range(1, n + 1):
            cur_max = 0
            best = 0
            for j in range(1, min(k, i) + 1):
                cur_max = max(cur_max, arr[i - j])
                best = max(best, dp[i - j] + cur_max * j)
            dp[i] = best
        return dp[n]

```
