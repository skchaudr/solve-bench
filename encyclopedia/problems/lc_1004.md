# lc_1004: Max Consecutive Ones III

**Topic**: [Sliding Window](../Sliding_Window.md)

**Empirical Time Complexity**: `O(N)`

**Empirical Space Complexity**: `O(1)`

## Notes

Time scales by a factor of ~10x as N increases by 10x, indicating O(N) time complexity. Peak memory usage remains near 0 KB across all scales, confirming O(1) space complexity.

## Solution

```python
from typing import List

class Solution:
    def longestOnes(self, nums: List[int], k: int) -> int:
        left = 0
        for right in range(len(nums)):
            # If we see a 0, we use one of our allowed flips
            k -= 1 - nums[right]

            # If we've used more flips than allowed, we need to slide the window forward
            if k < 0:
                # If the element sliding out of the window was a 0, we get a flip back
                k += 1 - nums[left]
                left += 1

        # The window size (right - left + 1) never shrinks, so its final size is the maximum
        return len(nums) - left

```
